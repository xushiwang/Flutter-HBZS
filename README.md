### 后台接口与架构
#### 后台接口

#### 架构方案1
> 采用官方接口:两台服务器能够在公网提供正常的接口服务,采用虚拟ip进行故障转移的处理,双机器安装keepalived
#### 架构方案2
> 内网自己建设服务器2台,需要具备可以访问内网资源的条件;两台web服务器部署在DMZ区域提供接口服务,需要可以访问公网,并且做nginx负载均衡实现内网机器的高可用,两外安装keepalived进行ip的漂移,实现高可以用.
